CSS: Cascading Style Sheet

Viết CSS ngay trong phần tử HTML này thì gọi là Inline CSS

    Nhược điểm

  1. Khó thay đổi, mỗi lần thay đổi phải làm thủ công cho từng dòng
  2. Gặp dòng nào có nhiều option thì nhìn lòi mắt luôn :V
  3. Không tái sử dụng được. Đây là một điều tối kỵ trong lập trình
  4. Khó quản lý
  5. Khó tách biệt cấu trúc

Internal CSS (CSS nội bộ) Là cách viết code CSS trong cặp thẻ style (Đặt trong khối head)

Ưu điểm

  1. Code tách biệt với phần tử, có thể áp dụng với toàn trang (Giống như ví dụ ta có thể áp dụng với toàn bộ thẻ h3)

Nhược điểm

  1. Khó quản lý khi dự án lớn, quá nhiều mã CSS sẽ làm tệp HTML trở nên cồng kềnh vì nó được viết trong thẻ head, có nhiều CSS sẽ làm rối loạn file HTML
  2. Nếu web có nhiều trang (Multi pages) thì ta phải lặp đi lặp lại đoạn mã css này trong các page đó gây ra không gọn gàng code
